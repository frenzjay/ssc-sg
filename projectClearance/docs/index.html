Clearance Track

Documentation

Overview

Clearance Tracker is a single-page application (SPA)

Requirements

  • Hardware:It can run on a Samsung Fridge
  • Software: Any modern web browser
  • Network: No internet connection required.

How?

  • Encoding ("Add Clearance")
    • Functionality: When you receives a batch of signed clearances, you will enter the Student Name and selects the corresponding Department from a dropdown menu.
    • Action: Clicking "+ Add to 'To Claim'" pushes the student's data to the top of the list and saves it to the local db.
  • Retrieval ("Search & Claim")
    • Search: A search bar filters the active list in as you type. Typing "Frenz" immediately hides all non-matching entries.
    • Department: Each entry displays a visible department badge directing the you the exact physical folder where the paper document is stored.
    • Claiming: Once the physical paper is handed to the student, you will click "CLAIM". The system says a dialog to prevent accidental deletions. Upon confirmation, the student is permanently removed from the queue.

Arch

  • localStorage: Every time an item is added or claimed, the JavaScript updates the browser's cache via JSON stringification, ensuring data survives reloads and closures.
